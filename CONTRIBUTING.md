# Contributing

Thank you for improving `@damngamerz/pi-otel`.

1. Open an issue before large behavioral or telemetry-schema changes.
2. Keep baseline telemetry free of conversation and tool content.
3. Add tests for lifecycle, privacy, and metric-label changes.
4. Run `npm run check` and `npm audit --omit=dev`.
5. Use concise semantic commit messages.

Report suspected vulnerabilities privately through GitHub security advisories rather than public issues.
